### auth
app 폴더 바로 아래에 위치하는 폴더. supabase 인증 관련 라우트를 모아둠.

#### supabase 는 google 계정의 패스워드를 모른다.
supabase 가 google oauth 인증을 통해 사용자의 계정을 생성하더라도, 해당 계정의
패스워드를 가져오지는 않는다. 따라서 이메일 + 비밀번호 로그인을 하려면 supabase 에서 사용하는 패스워드를 새로 설정해줘야 한다.

#### 패스워드를 모르는데 어떻게 구글 로그인에 성공할 수 있는가?
supabase 는 signInWithOAuth 메서드를 호출할 때, provider 값으로 google 을 넣어준다. 이는 google 에게 로그인하려는 사용자가 구글에 존재하는 실제 유저인지 확인해달라는 요청이다. supabase 는 google 의 답변에 따라 계정 생성 및 세션 발급을 진행한다. 즉 google oauth 방식은 패스워드를 필요로 하지 않는다. google 에서 패스워드를 입력해주고, supabase 구글에서 제공한 엑세스 토큰만을 이용하여 해당 사용자를 인증한다.

#### 그래서 내가 만들 기능은?
구글 회원가입을 통해 사용자의 계정을 생성. auth.updateUser 를 통해 비밀번호 재 설정. 이렇게 해주면 이메일 + 비밀번호 방식으로 로그인도 가능해짐.

## adminPage
관리자 관한을 가진 유저만 접근 가능. next js middle 를 이용하여 서버 측에서 강제 보호 수행.

### 기존 방법의 취약점
처음에 생각했던 관리자 판단 방법은 테이블의 role 필드 값을 검사하여 관리자를 판단하는 것이었다. 이 방법은 supabase.from.select 메서드를 통해 role 값을 확인하는 방식인데, 이는 개발자 도구에서 해당 값을 강제로 admin 으로 바꿔 로컬 상태를 조작할 우려가 있다. 즉 navbar 에서 role === admin 조건식으로 버튼을 보여주는 방식은 클라이언트에서 조작을 가할 경우, 얼마든지 보이게 할 수 있다.

### url 직접 접근 뚫림의 위험성
그냥 url 창에 /adminPage 이렇게 치면 바로 이동되기 때문에 이걸 막을 필요가 있다.

### RLS 정책만으로는 보호가 미약함.
관리자 페이지는 말그대로 관리자만 들어올 수 있게 해야하는데, supabase RLS 정책은 이 부분을 막아주지 못한다.

### middleware 의 필요성
next js 의 middleware 를 이용하면 url 직접 입력을 통한 접근을 막을 수 있다. middleware 는 요청이 서버에 도착하여 페이지가 실행되기 전에 가장 먼저 실행된다.
브라우저 -> URL 입력 -> next js 서버 -> 미들웨어 -> 관리자/일반 페이지
일반적으로 위와 같은 흐름으로 페이지 이동이 동작하는데, page.tsx 와 같이 페이지 코드들은 미들웨어 이후에 실행된다. 즉, 페이지 단에서 조작을 가하는 클라이언트 조작 방식은 미들웨어로 방어가 가능하다.

### middleware 는 뭐로 관리자를 판단하는가
일반적으로 쿠키나 헤더의 jwt 토큰을 이용한다. 하지만 단순히 쿠키 하나만 가지고 관리자를 판단하는 방법은 대단히 위험한 방법이다. 공유 pc의 경우, 관리자가 로그아웃을 하지 않고 그대로 페이지를 닫을 경우, 쿠키가 남아있을 가능성이 있다. 이 경우, 다음 사람이 url 을 통해 접근이 가능하면 미들웨어는 이를 관리자로 판단하기에 그대로 관리자 페이지가 노출된다. 이것이 바로 session persistence after logout, 로그아웃 이후 세션 지속성이라는 취약점으로 자주 지적되는 문제 중 하나이다.

### session persistence after logout 해결책
쿠키 하나만으로 판단하지 말고, 세션과 함께 동시 검증을 하거나 쿠키 옵션을 강화하는 방법이 있다.

### 새로운 해결책 : layout 이용하기
adminPage 의 layout 을 이용하여 관리자 계정이 접근했을 때만 관리자 페이지 및 하위 페이지를 로드하고, 다른 계정의 경우 홈 화면으로 리다이렉트 하는 방식을 적용해도 된다. 이렇게 하면, url 을 통해 관리자 페이지에 접근하는 방법을 막을 수 있다
